# Author frames as validated packs and require a fresh choice

Frames are data-driven Frame Pack content rather than bespoke compositing code. The Booth never auto-selects a Frame and returns to the picker after every completed capture so one guest's choice cannot carry into the next guest's photo.

## Consequences

Frame Pack validation and preview tooling may generate the runtime catalog, but capture timing and compositing remain driven by the selected Frame manifest. Operator convenience must not weaken the fresh-choice rule.
